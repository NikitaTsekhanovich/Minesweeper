# Minesweeper

### Комментарии по проекту

Конфиг для настройки поля: Assets/Configs/
#
Проект состоит из слоев: 
- Domain. Может мигрироваться в другие проекты. Не имеет зависимостей от других слоев. 
- Bootstrap. Точки входа в сцены или системы. Может иметь зависимости от Application.
- Application. Специфичные для проекта модули. Может иметь зависимости от Bootstrap и Domain.
- Presentation. Слой UI. Может иметь зависимости от всех других слоев.

Можно было строго зафиксировать зависимости через Assembly Definition, но в рамках тестового не счел необходимым.
#
Часть ивентов (например в GameTimer и Cell) можно было бы сделать через UniRx через реактивные свойства, что повысило бы читаемость и удобство, но по условию тестового использовать UniRx было нельзя. Свою реализацию в рамках тестового решил не делать, хоть базовый функционал и тривиальный.
#
Некоторые классы можно было бы сделать более универсальными, например, в конструктор Field прокинуть конкретные значения конфига, а не тянуть другие зависимости, что позволило бы переиспозовать класс в других проектах, но опять же, в рамках тестового не сочел это необходимым.
#
Можно было бы сделать выгрузку/загрузку блоков меню и игры через addressables, но в рамках тестового решил не усложнять.
